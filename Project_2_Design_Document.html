Project 2 Design Documentation

Requirements

Get a Makefile that compiles the two source files

    -I have the Makefile.

Have your main executable read in the command line arguments and output the values to your screen (and ensure -h displays useful results). Make sure that the parameters have correct values

Have master allocate shared memory, use it, then deallocate it. Make sure to check all possible error returns. Read the data into shared memory.

Get master to fork and exec one child and have that child attach to shared memory and read data from shared memory. Have the child output any information it was passed from master and then the values it read from shared memory. master should wait for it to terminate before terminating itself.

Put in the signal handling to terminate after specified number of seconds. A good idea to test this is to simply\par have the child go into an infinite loop so master will not ever terminate normally. Once this is working have it catch Ctrl-c and free up the shared memory, send a kill signal to the child and then terminate itself.

Set up the code to fork multiple child processes until the specific limits. You'll end up doing this in a nested loop. The outer loop will handle levels in the tree while the inner loop will take care of the pairs at the level.

Make each child process perform its share of the task and update the log file.

Ensure that the critical section problem is solved.
